Como funciona este monitor

📋 O que significa cada aba

Em Ambos A licitação foi publicada no PNCP e enviada ao APLIC. O cruzamento foi bem-sucedido.
Apenas PNCP Publicada no PNCP mas ainda não encontrada no APLIC. O município tem 3 dias úteis para enviar.
Apenas APLIC Existe no APLIC mas não foi ao PNCP. Pode indicar uma publicação irregular ou fora do prazo.
Alertas Ativos Licitações em "Apenas PNCP" cujo prazo de 3 dias úteis já venceu sem APLIC registrado.

🔗 Como o cruzamento é feito

O sistema tenta cruzar PNCP × APLIC em 3 etapas (tiers), do mais preciso ao mais amplo:

  1. Semântico + Valor — compara o texto do objeto/objetivo usando similaridade de texto (RapidFuzz) e verifica se o valor estimado diverge menos de 10%. É o critério mais confiável.
  2. CNPJ + Data — mesmo órgão (pelo CNPJ) e datas de publicação com até 30 dias de diferença. Usado quando o texto varia muito.
  3. Estrutural — mesmo município, número, ano e modalidade. Fallback de último recurso.

📊 O que é o Score

O score (0–100) mede a confiança no cruzamento. É calculado assim:

  • 50% — similaridade de texto (objeto PNCP vs objetivo/motivo APLIC)
  • 30% — proximidade de valor estimado
  • 20% — proximidade de data
≥ 85 → Confirmado 70–84 → Parcial < 70 → Sem match

🏛️ Os dois sistemas monitorados

  • PNCP (Portal Nacional de Contratações Públicas) — portal federal onde órgãos públicos são obrigados a publicar licitações. Dados coletados automaticamente via API.
  • APLIC (Sistema do TCE-MT) — sistema Oracle interno do Tribunal de Contas do Estado. Os municípios devem enviar as licitações em até 3 dias úteis após publicação no PNCP. Dados inseridos via exportação manual do Oracle.

Órgãos monitorados: Prefeitura Municipal de Sinop · Câmara Municipal de Sinop · IPREV Sinop
